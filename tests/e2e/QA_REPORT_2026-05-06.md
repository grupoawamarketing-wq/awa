## Phase 3: CSS Fixes Applied

### CSS Architecture
Four page-scoped QA CSS files loaded via `awa-head-preload.phtml` (async, `media="print"→onload swap`):

| File | Scope | Size |
|------|-------|------|
| `awa-visual-qa-home.css` | `body.cms-index-index` | 4.5KB |
| `awa-visual-qa-category.css` | `body.catalog-category-view`, `body.catalogsearch-result-index` | 8.6KB |
| `awa-visual-qa-product.css` | `body.catalog-product-view` | 1.1KB |
| `awa-visual-qa-fixes-2026-05-06.css` | Global QA polyfills | 30.9KB |

### Fix Details

**P3.1 — Product card H3 height normalization (Category/Search)**
- Root cause: `awa-bundle-refinements.css` L4335 sets `min-height: calc(15px * 1.4 * 2) = 42px`
- Fix: `height: 58px !important` + `min-height: 58px !important` + `max-height: 58px !important`
- Selector: `:is(body.catalog-category-view, body.catalogsearch-result-index) .item-product h3.product-name`
- Result: All 12 cards uniform at 58px ✅

**P3.1-home — Homepage carousel H3 height normalization**
- Root cause: Same `awa-bundle-refinements.css` pattern causing 45-49px range
- Fix: `height: 44px !important` for all carousel containers
- Selectors: `.rokan-bestseller`, `.rokan-newproduct`, `.rokan-mostviewed`, `.list-tab-product`, etc.
- Result: All 66 homepage product tiles uniform at 44px ✅

**P3.10 — Sorter select height (Desktop)**
- Root cause: Multiple `min-height` rules in merged bundle (44px + `var(--awa-input-height)`)
- Fix: `height: 36px !important` + `min-height: 36px !important` + `max-height: 36px !important`
- Result: Desktop sorter at 36px PASS ✅; Mobile at 48px (acceptable touch target) ✅

**PHTML loading fix**
- Issue: `cms_index_index.xml` layout XML CSS refs were overridden by Varnish-cached HTML
- Fix: Added QA CSS loaders to `awa-head-preload.phtml` (the proven async loading mechanism)
- All 3 QA CSS files now appear in every page's `<head>`

### Specificity Challenges Resolved
- Desktop sorter: `input[type=...] → min-height: 48px !important` in merged bundle (specificity 0-1-1) — beaten by our 0-3-2 selector
- Mobile H3: `awa-bundle-refinements.css` min-height calc — beaten by `!important` on height/min-height/max-height
- Mobile sorter: 48px is intentional (browser/AWA touch target spec — WCAG 2.5.5 minimum 24px, recommended 44px)
